### 第一档（MUST / 高价值·低风险·无需实测，建议优先）
| 命令 | 价值 | 备注 |
|---|---|---|
| **`project-save`** ✅已补 | 写命令闭环关键 | `project.Save()`，否则改动丢内存 |
| `project-info` | 连接后第一手上下文 | 纯读 Project.Name/Author/Path/Version/时间/注释(MultilingualText) |
| `device-list` | 工程拓扑入口 | 遍历 Project.Devices + UngroupedDevicesGroup，区分 PLC/HMI/其他 |
| `block-info <块>` | 屎山排查/重构判断 | 块号/语言/CompileDate/ModifiedDate/IsConsistent/作者/版本/MemoryLayout（**无"大小"**，手册无该 API） |

### 第二档（SHOULD / 可行高价值，多为只读或增强）
- `device-info`(型号/订货号/固件——CPU DeviceItem 属性) · `device-modules`(机架槽位树) · `device-network`(IP/子网)
- `rename-block`(SetAttribute Name，**改名不更新引用**，须告警 + 编译核对) · `delete-tags`(PLC 变量删) · `read-constants`(用户/系统常量)
- `library-list`(项目库/全局库只读枚举) · 交叉引用**报表导出**(聚合 where-used → CSV/MD，内存交付)
- 项目级 `ExportProjectTexts/ImportProjectTexts`(一次性导全工程可翻译文本到 xlsx 批改，含块/网络注释 + HMI 文本)

### 第三档（NICE / 低频或带副作用，按需）
- `set-block-number` · `edit-tag`(并入 write-tags upsert) · `export-watchtable`/`export-forcetable`(表定义非活值)
- `compare-software`(PLC↔库/在线，签名待编译核实) · 项目归档 `.zap18` · `check-address-overlap`(仅 %I/%Q/%M，信息非错误)
- `plc-go-online`/`plc-online-state`(**仅连接态**，非 RUN/STOP) · `download-plc`(去掉 --software-only；停机类默认二次确认；现场红线)
- `hmi-import-list`/`hmi-delete-list`(写文本/图形列表)

### 已否决（对抗核实 refuted/drop）
- ❌ `read-db`：**已被 export-xml 覆盖**（DB 是 PlcBlock，无语言门槛，XML 含 `<Interface><Member>`）。最多做薄包装解析，降为 could。
- ❌ `hmi-export-lists`：**已被 hmi-export-all 覆盖**（读/导出列表）。真缺口是"写列表"。
- ❌ `create-block-group`：能建空组但 SCL 块只能落根组（手册 15451），"移块进组"无法闭环。
- ❌ `library-make-mastercopy`：与"单项目、交付物=内存文本"哲学不符，收益低于 export/import 文本往返。
- ⏸ `project-manifest`：MCP 阶段编排 AI 直接串 list+read-tags+read-udts+hmi-list 即可，零新代码。
- ⏸ 写命令 `--device` 选项：单设备工程零收益；真做应下沉为 TiaSession.ResolvePlc/ResolveHmi 供读写共用。

## 3. Openness 手册优化机会（全部经手册行号求证）

| 机会 | 现状 | 建议 | 置信 |
|---|---|---|---|
| **原生交叉引用** | 导出文本+扫描 | **维持**——V18 全文 0 命中 `CrossReference/GetCrossReferenceInfo/…`，无可替代品。结论固化、文档注明"内容扫描·非语义级" | high |
| **批量属性读** | 逐属性 GetAttribute + try/catch | 用 `((IEngineeringObject)o).GetAttributes(names)` 一次 COM 往返取齐（list/read-tags 提速）。**不要每块先 GetAttributeInfos 探测**（反增往返）；对 know-how 块特判兜底 | high |
| **块元数据属性** | list 只给类型/语言 | `block-info` 一并读 Number/MemoryLayout/Header*/各 Date/IsConsistent（手册 15119/27162） | high |
| **编译结果过滤** | PrintMessages 打全部 message | 按 `State==CompilerResultState.Error` 过滤（MCP 省 token）；**始终递归下钻**、只在叶子按 State 决定输出；退出码逻辑不变 | high |
| **块/变量注释** | 完全没读（SCL 注释还被剥离） | `read-tags` 增读 `tag.Comment`(MultilingualText `.Items.Find(lang).Text`，优先 zh-CN)。**块注释 CommentML 需活机实测**（手册仅 device 示例，且返回 Item 单数） | high(tag)/med(block) |
| **导出体积** | PLC 用 None / HMI 用 WithDefaults | **维持**：None 已最精简（手册 25950）；HMI 经典对象需 WithDefaults 才完整。block-deps 可选 `GenerateOptions.WithDependencies` 拿"块+依赖" | high |
| **按名查块** | 自写递归遍历 | 单层可先 `group.Blocks.Find(name)` 命中即返回（Find 非递归，跨子组仍递归）。**须保留大小写不敏感兜底**（Find 大小写行为手册未保证） | med |
| **批量写属性** | write-tags 用 Create | 未来"改属性"类命令用 `SetAttributes(同对象多属性)`（单事务回滚）；ProDiag AssignedProDiagFB 退回单属性；类型不支持的属性 catch `EngineeringNotSupportedException` | med |

### 未修复（已记录，归路线图或低优先）
- **LooksEncrypted 对 UTF-16 输入误判密文**（low）：手工另存 UTF-16 的明文 XML 会被拒。MCP 阶段走 inline UTF-8 可绕开；要修需先识别 BOM 再解码。
- **交叉引用每次全量导出**（low 性能）：CLI 下每命令独立进程无法跨命令缓存；**MCP 长驻进程应缓存"块名→导出文本/引用集合"索引**（关键 MCP 设计点）。
- **空图形块被判受保护**（low 诊断）：无 CompileUnit 的合法空块会进"受保护"列。保守不致误删，仅诊断措辞误导。
- **import-scl 覆盖同名块无 dry-run/确认**（low/med）：建议解析源中块名比对、命中既有先告警。
- **where-used <FB> 漏 SCL/iDB 调用方**（与 #1 同根）：完整修法需反查 InstanceOfName==FB 的 iDB 再找其引用者；当前仅修了更危险的 find-unused 方向。
- **AttachFirst 多实例选择**：仅加了诊断打印；按项目名/路径选实例待 MCP 入参。
